# Speckit 프로젝트 기능 명세 구체화를 위한 질문 리스트 (Clarification Questions)

`specification.md`를 바탕으로, 실제 구현 및 정책 수립을 위해 결정이 필요한 사항들을 정리했습니다.

---

## 1. 사용자 관리 (Auth)
- **Q1.1. JWT 만료 정책**: Access Token과 Refresh Token의 구체적인 만료 시간은 어떻게 설정할까요? (예: Access 30분, Refresh 2주)
- **Q1.2. 소셜 로그인 범위**: 초기 MVP 단계에서 Google, GitHub 외에 필수적으로 포함해야 할 소셜 프로바이더가 있나요? (예: Kakao, Naver)

## 2. 데이터 분석 및 시각화
- **Q2.1. 파일 업로드 제한**: 사용자당 업로드 가능한 최대 파일 크기나 스토리지 용량 제한을 어느 정도로 두시겠습니까? (예: 파일당 10MB, 총 100MB)
- **Q2.2. 지원 파일 형식**: CSV, Excel, PDF 외에 지원해야 할 필수 포맷이 있나요? (예: JSON, Parquet)

## 3. CDTV (데이터-투-비디오)
- **Q3.1. 비디오 생성 API**: Synthesia와 HeyGen 중 우선적으로 연동할 타겟 서비스가 있으신가요? 혹은 두 서비스 모두를 지원하는 추상화 계층이 필요하신가요?
- **Q3.2. 아바타/보이스 설정**: 사용자별로 커스텀 아바타/보이스를 등록할 수 있게 할까요, 아니면 시스템이 제공하는 프리셋만 사용하게 할까요?

## 4. SNS 자동 포스팅
- **Q4.1. YouTube 승인**: YouTube Data API의 `upload` 권한은 Google의 까다로운 앱 검수를 필요로 합니다. 개발 단계에서는 테스트 계정으로만 진행하시겠습니까?
- **Q4.2. 추가 채널**: 2순위로 고려 중인 SNS 채널이 있다면 무엇인가요? (아키텍처 설계 시 고려)

## 5. 구독 및 결제
- **Q5.1. 결제 PG사**: 글로벌 결제(Stripe)를 타겟으로 하시나요, 아니면 국내 PG(토스페이먼츠 등)를 타겟으로 하시나요?
- **Q5.2. 무료 플랜 제한**: Free 티어의 "월 n회 영상 생성"에서 n의 구체적인 수치는? (API 비용 고려 필요)

## 6. 인프라 및 배포
- **Q6.1. 배포 타겟**: 최종 배포 환경이 AWS, GCP, Azure, 혹은 온프레미스 서버 중 어디인가요?
- **Q6.2. 도메인**: 서비스에 연결할 도메인이 이미 준비되어 있나요?

---

위 질문들에 대한 답변을 주시면, `specification.md`를 업데이트하고 상세 구현 계획(Implementation Plan)을 수립하겠습니다.
