# GC101

## 1. Why (목적)
현대 조직의 데이터는 파편화되고 복잡합니다. 사용자들은 단순히 문서를 "검색"하는 것을 넘어, 데이터 간의 **관계를 이해**하고 **트렌드를 분석**할 수 있는 방법이 필요합니다.

GC101은 사람의 의도와 복잡한 데이터 시스템 사이의 간극을 좁히기 위해 존재합니다. 우리의 목표는 사용자가 전문 지식 없이도 마치 데이터 분석 전문가와 대화하듯이 데이터와 상호작용하여, 원시 정보를 실행 가능한 통찰력으로 변환할 수 있도록 돕는 것입니다.

## 2. How (아키텍처)
우리는 **LangGraph**에 기반한 선진적인 **에이전트형 AI 아키텍처**를 통해 이를 달성합니다:

- **오케스트레이션 (LangGraph)**: 그래프 기반의 상태 머신 아키텍처를 활용합니다. 중앙의 "Router"가 사용자의 의도를 지능적으로 파악하여 전문화된 노드로 연결합니다:
    - **General Node**: 일반적인 대화 및 인사말 처리.
    - **RAG Node**: 심층적인 정보 검색 수행.
    - **Analyst Node**: 정형 데이터 처리 및 분석 (로드맵 예정).
- **하이브리드 검색 (GraphRAG)**: 두 가지 강력한 데이터베이스 기술을 결합합니다:
    - **Neo4j (Graph DB)**: 데이터 간의 구조적 관계를 포착합니다 (누가 무엇을 작업했나? 이것이 누구와 연결되나?).
    - **Qdrant (Vector DB)**: 정확한 키워드 일치가 없어도 의미론적 벡터 검색을 통해 관련 개념을 찾아냅니다.
- **현대적인 기술 스택**: 
    - **Backend**: Python **FastAPI**를 사용하여 고성능 비동기 기반을 제공합니다.
    - **Frontend**: **Nuxt 3**를 사용하여 반응형 서버 사이드 렌더링(SSR) 경험을 제공합니다.
    - **가시성(Observability)**: 포괄적인 로깅 시스템을 통해 모든 에이전트의 의사결정 과정을 추적할 수 있습니다.

## 3. What (제품)
GC101은 다음과 같은 기능을 갖춘 **AI 기반 데이터 플랫폼**입니다:

- **문맥 인식 챗봇**: 대화 상태를 유지하고 사용자의 의도를 정확히 파악하는 지능형 비서입니다.
- **지식 발견**: 지식 그래프(Knowledge Graph)와 벡터 저장소(Vector Store)의 정보를 종합하여 복잡한 질문에 답합니다.
- **확장 가능한 기반**: **Docker**를 통해 완전한 컨테이너 환경을 제공하여 배포가 쉽고 확장이 용이합니다.
